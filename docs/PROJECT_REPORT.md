# Отчёт: шаблонный репозиторий agency-boilerplate

**Контекст.** Репозиторий в организации **Qascade-AGC** подготовлен с нуля как **GitHub Template** для старта клиентских и внутренних проектов агентства (лендинги, админки, внутренние инструменты, MVP).

**Репозиторий:** https://github.com/Qascade-AGC/agency-boilerplate  

---

## GitHub и процесс

- Зафиксированы ожидаемые практики для организации: чеклист `.github/ORGANIZATION_SETUP.md`, политика уязвимостей `SECURITY.md`.
- Репозиторий включён как **Template repository** (создание новых проектов через *Use this template*).
- Настроены **шаблоны Issues** (баг и задача/идея) и **шаблон Pull Request** (`.github/`).
- Для ветки **`main`** используется защита изменений через **Pull Request** (по политике команды).
- Подключён **Dependabot** для зависимостей npm (`.github/dependabot.yml`).
- Добавлен **CI** в GitHub Actions: на push и PR в `main` выполняются `npm ci` и production-сборка `npm run build` (`.github/workflows/ci.yml`). Workflow можно указать как обязательный статус в branch protection.

---

## Технический стек и функциональность приложения

- **Next.js 15** (App Router), **React 19**, **TypeScript**, **Tailwind CSS 4**.
- **PostgreSQL** и **Prisma**: схема, миграции в `prisma/migrations`; служебная таблица для демонстрации работы БД; таблицы для **Auth.js / NextAuth** (пользователи, аккаунты, сессии и связанные сущности).
- Эндпоинт **`GET /api/health`** и индикатор состояния БД на главной странице.
- **Auth.js (NextAuth v5)** с провайдером **GitHub OAuth**: страница `/login`, защищённая зона `/dashboard`, **middleware** без доступа без сессии; конфигурация с учётом совместимости с Edge (отдельный `auth.config.ts`).
- Учтена интеграция с **Neon на Vercel**: резервное использование типичных имён переменных окружения для строки подключения к БД; ответы API health с корректной кодировкой UTF-8.
- Базовый **UI-kit** в `components/ui` (кнопка, карточка, поля ввода, бейдж и др.); тема и CSS-переменные для стабильного отображения интерфейса.

---

## Инфраструктура и документация

- Локальная БД: **docker-compose** с PostgreSQL; в README описан порядок работы с `.env` и миграциями.
- **Деплой на Vercel** от репозитория GitHub; в README задокументированы переменные окружения (БД, `AUTH_SECRET`, ключи GitHub OAuth, callback URL для локальной среды и production).

---

## Итог

Подготовлен **единый стартовый шаблон**: репозиторий как template на GitHub, процесс (шаблоны задач и PR, защита ветки, автопроверка сборки, Dependabot), приложение с подключением к БД, проверкой работоспособности, входом через GitHub и базовым UI. При корректно заданных переменных в Vercel и в настройках GitHub OAuth решение готово к использованию как основа для новых кейсов и клиентских проектов.
